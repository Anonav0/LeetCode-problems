# 🔹 LC 203 — Remove Linked List Elements

### 🧠 Problem

Given:

- `head` of linked list
- `val`

👉 Remove all nodes with value = `val`

---

# 🔴 Step 1: Naive Thinking

Traverse and delete nodes… but:

👉 Problem:

- What if **head itself needs to be removed?** ❌

---

# 🔥 Step 2: Key Trick — Dummy Node

👉 This is the **real pattern**

We add a fake node before head:

```text
dummy → head → ...
```

---

# 🔹 Why Dummy Helps

- Uniform handling of all nodes
- No special case for head

---

# 🔹 Step 3: Pointer Setup

- `prev` → points to last valid node
- `curr` → traverses list

---

# 🔹 Step 4: Logic

If:

```text
curr.val == val
```

👉 skip it:

```text
prev.next = curr.next
```

Else:

```text
move prev forward
```

---

# ✅ Java Code

```java
class Solution {
    public ListNode removeElements(ListNode head, int val) {
        ListNode dummy = new ListNode(-1);
        dummy.next = head;

        ListNode prev = dummy;
        ListNode curr = head;

        while (curr != null) {
            if (curr.val == val) {
                prev.next = curr.next; // remove
            } else {
                prev = curr; // move prev only if not removed
            }
            curr = curr.next;
        }

        return dummy.next;
    }
}
```

---

# 🔥 Dry Run (Important)

```text
head = [1,2,6,3,4,5,6], val = 6

dummy → 1 → 2 → 6 → 3 → 4 → 5 → 6
```

- remove first 6 → link 2 → 3
- remove last 6 → link 5 → null

---

# ⚡ Complexity

- Time: `O(n)`
- Space: `O(1)`

---

# 🔥 Pattern Recognition

👉 This is:

> **Linked List Deletion Pattern with Dummy Node**

---

# 🔹 Where Else This Appears

| Problem             | Pattern |
| ------------------- | ------- |
| Remove elements     | dummy   |
| Remove nth from end | dummy   |
| Merge lists         | dummy   |
| Partition list      | dummy   |

---

# 🧠 Key Insight

👉 Always ask:

```text
Will head change?
```

If yes → use **dummy node**
