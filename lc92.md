# 🔹 LC 92 — Reverse Linked List II

### 🧠 Problem

Reverse a linked list from position `left` to `right`

---

### Example

```text
1 → 2 → 3 → 4 → 5
left = 2, right = 4

Output:
1 → 4 → 3 → 2 → 5
```

---

# 🔴 Step 1: What’s Different from LC206?

👉 LC206:

- Reverse entire list

👉 LC92:

- Reverse **only a segment**
- Then reconnect properly

---

# 🔥 Step 2: Key Idea

Break the problem into 3 parts:

```text
(before) → [reverse part] → (after)
```

---

# 🔹 Step 3: Important Pointers

We need to track:

- `prev` → node before `left`
- `curr` → start of reversal
- `tail` → original `left` (will become end after reverse)

---

# 🔹 Step 4: Algorithm

### 1. Use dummy node

```text
dummy → head
```

---

### 2. Move `prev` to `(left - 1)`

---

### 3. Reverse `right - left + 1` nodes

---

### 4. Reconnect

---

# ✅ Java Code (Standard Solution)

```java
class Solution {
    public ListNode reverseBetween(ListNode head, int left, int right) {
        if (head == null || left == right) return head;

        ListNode dummy = new ListNode(0);
        dummy.next = head;

        // Step 1: reach node before left
        ListNode prev = dummy;
        for (int i = 1; i < left; i++) {
            prev = prev.next;
        }

        // Step 2: start reversal
        ListNode curr = prev.next;

        for (int i = 0; i < right - left; i++) {
            ListNode next = curr.next;

            curr.next = next.next;
            next.next = prev.next;
            prev.next = next;
        }

        return dummy.next;
    }
}
```

---

# 🔥 Key Trick (HEAD INSERTION)

This line is everything:

```java
next.next = prev.next;
prev.next = next;
```

👉 You are inserting nodes at the front of the sublist

---

# 🔹 Visualization (Critical)

Initial:

```text
prev → 2 → 3 → 4 → 5
        ↑
      curr
```

After one iteration:

```text
prev → 3 → 2 → 4 → 5
```

👉 You keep pulling `next` to the front

- Time: `O(n)`
- Space: `O(1)`
