# 🔹 LC 77 — Combinations

### 🧠 Problem

Given:

- `n` → numbers from `1 to n`
- `k` → size of subset

👉 Return all combinations of size `k`

---

### Example

```text
n = 4, k = 2

Output:
[ [1,2], [1,3], [1,4], [2,3], [2,4], [3,4] ]
```

---

# 🔴 Step 1: Connect to Previous Problem

👉 This is just **subsets with constraint = size k**

So:

- Same backtracking
- Add condition: only take size `k`

---

# 🔥 Step 2: Core Idea

We generate numbers from `1 → n`

Track:

- `start` → where to pick next
- `path` → current combination

---

# 🔹 Step 3: Base Case

```text
if path.size() == k → add result
```

---

# ✅ Java Code (Clean Version)

```java
class Solution {
    public List<List<Integer>> combine(int n, int k) {
        List<List<Integer>> result = new ArrayList<>();
        backtrack(1, n, k, new ArrayList<>(), result);
        return result;
    }

    private void backtrack(int start, int n, int k, List<Integer> path, List<List<Integer>> result) {
        if (path.size() == k) {
            result.add(new ArrayList<>(path));
            return;
        }

        for (int i = start; i <= n; i++) {
            path.add(i);                         // choose
            backtrack(i + 1, n, k, path, result); // explore
            path.remove(path.size() - 1);        // undo
        }
    }
}
```

---

# 🔥 Key Insight

👉 `start` ensures:

- No duplicates
- No reusing same number
- Maintains order

---

# ⚡ Complexity

- Time: `O(C(n, k) * k)`
- Space: `O(k)`

---

# 🔥 Optimization (VERY IMPORTANT — Interview Level)

We can **prune unnecessary branches**

👉 Suppose:

- remaining numbers < needed numbers → stop early

---

### Optimized Loop

```java
for (int i = start; i <= n - (k - path.size()) + 1; i++)
```

---

### Why?

👉 Ensure enough elements remain to fill `k`

---

# 🧠 Pattern Connection

| Problem      | Idea          |
| ------------ | ------------- |
| Subsets      | all sizes     |
| Combinations | fixed size k  |
| Permutations | order matters |
