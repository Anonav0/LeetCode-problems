## 📜 Problem Statement

Given the array nums, for each nums[i] find out how many numbers in the array are smaller than it. That is, for each nums[i] you have to count the number of valid j's such that j != i and nums[j] < nums[i].

Return the answer in an array.

Example 1:

Input: nums = [8,1,2,2,3]
Output: [4,0,1,1,3]
Explanation:
For nums[0]=8 there exist four smaller numbers than it (1, 2, 2 and 3).
For nums[1]=1 does not exist any smaller number than it.
For nums[2]=2 there exist one smaller number than it (1).
For nums[3]=2 there exist one smaller number than it (1).
For nums[4]=3 there exist three smaller numbers than it (1, 2 and 2).

Example 2:

Input: nums = [6,5,4,8]
Output: [2,1,0,3]

Example 3:

Input: nums = [7,7,7,7]
Output: [0,0,0,0]

Constraints:

    2 <= nums.length <= 500
    0 <= nums[i] <= 100

---

## 🔍 Key Observations

- can be solved through brute force On^2 solution.
- can be solved using hashmap by storing the sorted array and index.
- constaint `0 <= nums[i]<= 100`; this small range allows counting technique. (prefix solution)

## 💡 Approach

### Approach 1: Brute Force

Compare each number with every other number.

**Steps**

```
for each i
   count = 0
   for each j
        if nums[j] < nums[i]
             count++
```

**Time Complexity:** O(n^2)  
**Space Complexity:** O(1)

```java
class Solution {
    public int[] smallerNumbersThanCurrent(int[] nums) {
        // ArrayList<Integer> result = new ArrayList<Integer>();

        int count =0;
        int n = nums.length;
        int res[] = new int[n];
        for(int i=0; i<n;i++){
            for(int j=0; j<n; j++){
                if(nums[j]< nums[i]){
                    count++;
                }
            }
            // result.add(count);
            res[i] = count;
            count=0;
        }
        return res;
    }
}
```

### Approach 2: Sorting + HashMap

Idea

Sort the array so we know the position of each number.

The first index of a number in the sorted array tells how many numbers are smaller.

Example:

nums = [8,1,2,2,3]

sorted = [1,2,2,3,8]

Create map:

1 → 0
2 → 1
3 → 3
8 → 4

Meaning:

value → count of smaller numbers

Then map back to original array.

```java
import java.util.Arrays;
class Solution {
    public int[] smallerNumbersThanCurrent(int[] nums) {
        int temp[] = nums.clone();
        Arrays.sort(temp);
        HashMap<Integer,Integer> map = new HashMap<>();
        int n = nums.length;
        int res[] = new int[n];
        for(int i=0; i<n;i++){
            if(!map.containsKey(temp[i])) {
                map.put(temp[i],i);
            }
        }
        for(int i=0; i<n;i++){
            res[i]=map.get(nums[i]);
        }
        return res;
    }
}
```

**Time Complexity:** O(n log n)  
**Space Complexity:** O(n)

### Approach 3 : Counting Arrays (Prefix sum + frequency)

Because numbers are limited:

0 ≤ nums[i] ≤ 100

We can count frequencies.

**Idea**

Create frequency array:

count[101]

Step 1 — count occurrences

count[num]++

Step 2 — prefix sum

count[i] += count[i-1]

Step 3 — smaller numbers:

count[num-1]

```java
class Solution {
    public int[] smallerNumbersThanCurrent(int[] nums) {

        int[] count = new int[101];

        for(int num : nums)
            count[num]++;

        for(int i = 1; i < 101; i++)
            count[i] += count[i-1];

        int[] result = new int[nums.length];

        for(int i = 0; i < nums.length; i++){
            if(nums[i] == 0)
                result[i] = 0;
            else
                result[i] = count[nums[i]-1];
        }

        return result;
    }
}
```

**Time Complexity:** O(n)  
**Space Complexity:** O(1)
