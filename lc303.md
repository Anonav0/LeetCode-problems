Good — now we enter **Prefix Sum**, one of the most powerful tools in arrays.

# 🔹 LC 303 — Range Sum Query (Immutable)

### 🧠 Problem

You are given an array `nums[]`

You need to answer queries:

```text
sumRange(left, right)
```

👉 Return sum from index `left` to `right` (inclusive)

---

# 🔴 Step 1: Brute Force

For every query:

- Loop from `left → right`

👉 Time per query = `O(n)` ❌
👉 Total = very slow if many queries

---

# 🔥 Step 2: Key Idea — Precompute

Instead of recomputing again and again:

👉 Store cumulative sums

---

# 🔹 Step 3: Prefix Sum Definition

```text
prefix[i] = sum of elements from index 0 to i
```

Example:

```text
nums = [1,2,3,4]

prefix = [1,3,6,10]
```

---

# 🔥 Step 4: How Query Becomes O(1)

To find:

```text
sum(left, right)
```

👉 Use:

```text
prefix[right] - prefix[left - 1]
```

---

# 🔹 Visualization

\text{sum}(l,r)=prefix[r]-prefix[l-1]

---

# 🔹 Edge Case

If `left == 0`:

```text
sum = prefix[right]
```

---

# 🔹 Step 5: Code Design (IMPORTANT)

This is not just a function — it’s a **class problem**

- Precompute in constructor
- Answer queries in `O(1)`

---

# ✅ Java Code

```java
class NumArray {
    private int[] prefix;

    public NumArray(int[] nums) {
        prefix = new int[nums.length];

        prefix[0] = nums[0];
        for (int i = 1; i < nums.length; i++) {
            prefix[i] = prefix[i - 1] + nums[i];
        }
    }

    public int sumRange(int left, int right) {
        if (left == 0) return prefix[right];
        return prefix[right] - prefix[left - 1];
    }
}
```

---

# ⚡ Complexity

| Operation   | Time |
| ----------- | ---- |
| Constructor | O(n) |
| Query       | O(1) |

---

# 🔥 Key Insight

👉 You trade:

- **Extra space + preprocessing**
  for
- **fast queries**

---

# 🧠 Pattern Recognition

This pattern is used everywhere:

- Range sum queries
- Subarray sum problems
- Sliding window optimizations
- 2D prefix sums
